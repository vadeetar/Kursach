# Сценарий демонстрации на защите (5–7 минут)

## Подготовка (до защиты)

```bash
cd E:\курсач
docker compose up -d --build
# Проверка: http://localhost:8000/api/v1/health
```

Опционально: в `.env` указать `NVD_API_KEY` для быстрой синхронизации.

## Шаг 1 — Веб-интерфейс (1 мин)

1. Открыть http://localhost:8000  
2. Вкладка **Дашборд** — показать счётчики CVE, активов, находок.  
3. Вкладка **Находки** — фильтр `priority=critical`, просроченные SLA.

## Шаг 2 — Управление исправлением (1 мин)

1. Нажать **Изменить** у находки.  
2. Статус: `in_progress`, исполнитель: `platform`, заметка: «обновление пакета».  
3. Сохранить — показать обновление в таблице.

## Шаг 3 — Инвентарь (1 мин)

1. Вкладка **Инвентарь** → добавить пакет `flask` / `3.0.0` / `pypi`.  
2. Показать появление в списке.

## Шаг 4 — API и NVD (2 мин)

1. Открыть http://localhost:8000/api/v1/docs  
2. Показать `GET /api/v1/vulnerabilities`, `GET /api/v1/reports/executive-summary`  
3. Вкладка **Синхронизация** → **Синхронизировать NVD** (нужен интернет)  
4. Кратко: данные из NIST NVD API 2.0, EPSS, сопоставление активов.

## Шаг 5 — Качество и репозиторий (1 мин)

1. GitHub: https://github.com/vadeetar/Kursach  
2. Показать `.github/workflows/ci.yml` — тесты, Bandit, Ruff  
3. `backend/tests/` — автотесты  

## Возможные вопросы комиссии

| Вопрос | Ответ |
|--------|--------|
| Что такое CVSS? | Базовая оценка серьёзности уязвимости 0–10 (NVD). |
| Зачем EPSS? | Вероятность появления эксплойта — дополняет CVSS. |
| Как сопоставляете CVE и пакет? | CPE/описание + имя пакета + версия (`matcher.py`). |
| Почему FastAPI? | Async, OpenAPI, Pydantic, высокая скорость разработки. |
| Где хранятся данные? | PostgreSQL (Docker), модели SQLAlchemy. |
