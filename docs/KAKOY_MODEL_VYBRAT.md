# Какой режим / модель Cursor выбрать для курсовой

## Для написания и доработки текста курсовой в Word

| Режим / модель | Рекомендация |
|----------------|--------------|
| **Opus 4.8 High** | Лучший выбор для длинного связного текста, анализа, структуры глав |
| **GPT-5.5 Medium** | Хорошая альтернатива для объёмных документов |
| **MAX Mode** | Включите, если нужен максимум качества и глубины ответов |
| **Sonnet 4.6 Medium** | Нормально для правок, но для «идеальной» полной курсовой слабее Opus |
| **Composer 2.5 Fast** | Только для кода, не для 30+ страниц текста |
| **Auto Efficiency** | Экономит ресурсы, не для финальной версии записки |

## Практический совет

1. **Генерация полного DOCX** — уже сделана скриптом `generate_kursovaya_full.py` (не нужна модель).
2. **Доработка формулировок, вуз, рецензия** — переключитесь на **Opus 4.8 High** + **MAX Mode**.
3. **Код, тесты, GitHub** — **Composer** или **Sonnet**.

## Промпт для доработки в Opus

```
Доработай раздел [номер] курсовой по платформе управления уязвимостями (вариант 4).
Стиль: академический русский, Times 14, без воды.
Проект: FastAPI, NVD, EPSS, GitHub vadeetar/Kursach.
Добавь 2–3 абзаца про [тема раздела].
```
